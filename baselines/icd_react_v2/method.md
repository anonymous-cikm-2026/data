The Core Toolkit 

There are three primary documents used in the coding process: 

Alphabetic list: The starting point. It is an alphabetical lookup used to find a base code (or range) from a clinical condition or symptom. 

Tabular list: A numerical list of all codes. It is used to verify the code found in the Index and shows the hierarchy (e.g., specific complications or sub-types of a disease). 

Official Guidelines: The "Logic Engine." This book contains the mandatory rules for coding, including General Rules (sequencing) and Chapter-Specific Rules (logic for specific diseases like Diabetes or Sepsis). 

High-Level Coder Workflow 

For every piece of medical information in a record, the coder follows this deterministic path: 

Identify & Search: Look up the condition in the Alphabetic Index to get a preliminary code. 

Verify & Specify: Locate that code in the Tabular List. This confirms the code is valid and checks if more digits are needed for higher specificity. 

Reference Rules: Check the Official Guidelines for any logic that applies to that specific category of disease. 

Aggregate & Filter: Collect all potential codes and apply the guidelines to create a final, compliant list. For e.g., if one of the guidelines says don’t code both usage of drugs, and dependence on drugs, then only retain the codes related to dependence on drugs.
